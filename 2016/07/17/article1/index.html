简单地聊一聊博客的搭建过程

  • Hexo,Git,Node.js

博客搭建的方式网上有很多，因为小菇凉比较熟悉Github，于是乎用Hexo来搭建了本博客。网上有很多详细的教程，小菇凉也看了很多教程，在这里划一下重点，也顺带讲一下自己遇到的问题。

1.安装环境

去官网下载Git，Node.js，并安装。

2.建立仓库

在自己的Github上建立一个仓库(repository)，仓库名一定要是 【your_user_name.github.io】。如果没有Github账号，直接用邮箱注册一个就行了，很方便。

3.配置SSH-key

这一步主要是用来连接本地仓库的文件夹与Github仓库的。网上的教程很多也很全面，小菇凉在这里就不赘述了。但要记住一点，如果遇到以下报错

可以通过先 pull，再 merge 冲突，再 push 的方法来解决。
当然如果还是不行，又百分百保证没有其他问题的情况下，不妨试一下

强制 push 。这种方法有点强盗，不提倡，但倒也是能解决问题的。

4.安装Hexo

在之前的本地仓库的文件夹下右键打开Git Bash

安装完毕后进行部署

完成后，根据提示，在浏览器中打开 localhost:4000 就能在本地看到自己的博客啦~
然后可以根据自己的喜好，去网上找一些自己喜欢的主题替换掉那个丑陋的原始主题，在本地仓库的文件夹中找到 config 文件，修改其中关于主题的配置。

5.绑定自己专属域名

在本地仓库中找到themes文件夹，打开后找到自己现在正在用的主题，找到其中的 config 文件，将其中的 url 替换成你自己的专属域名保存即可。

附：Hexo常用操作

简写

最后再多说一句，所有的 config 文件的配置项，冒号后面都必须加一个空格！！

其实博客的搭建过程并不复杂，细心地走好每一步就行了。
